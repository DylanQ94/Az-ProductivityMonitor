# Project Overview

## Objective

The objective of the Productivity Monitor Lakehouse on Azure project is to build an end-to-end data engineering pipeline for productivity monitoring data generated by a local desktop application.

The solution moves raw CSV and JSON files from a local machine into Azure Data Lake Storage Gen2, processes the data with Azure Databricks and Delta Lake, and exposes curated Gold data through Azure Synapse Analytics Serverless SQL views.

## Business Context

The source application monitors productivity-related activity while a user works on a PC. It produces daily files containing activity signals, cumulative duration counters, and activity category information.

The analytical goal of the project is to convert those raw daily files into a reliable analytical model that can answer productivity questions based on real duration metrics rather than raw cumulative counters.

## Current Project Scope

The current documented scope covers:

- Local file ingestion with Azure Data Factory.
- Local source access through Self-hosted Integration Runtime.
- Storage in Azure Data Lake Storage Gen2.
- Bronze, Silver, and Gold processing with Azure Databricks.
- Delta Lake analytical modeling.
- Synapse Serverless SQL views over Gold Delta folders.

## Main Components

| Component | Role |
|---|---|
| Local Productivity Monitor application | Generates daily CSV or JSON productivity files. |
| Azure Data Factory | Orchestrates ingestion from the local folder into ADLS Gen2. |
| Self-hosted Integration Runtime | Allows ADF to access the local file system. |
| Azure Data Lake Storage Gen2 | Stores Landing, Bronze, Silver, and Gold data. |
| Azure Databricks | Processes raw files into curated Delta Lake tables. |
| Delta Lake | Provides the storage format for processed lakehouse layers. |
| Azure Synapse Serverless SQL | Exposes Gold Delta folders through SQL views. |

## Source Data

Confirmed local source folder:

```text
C:\ProductivityMonitor\Reports
```

The application produces daily files in either CSV or JSON format.

Original source columns:

```text
timestamp
level
face_front
input_active
session_seconds
attentive_seconds
productive_seconds
activity
```

## Final Analytical Output

The final Gold layer contains fact and dimension Delta tables designed for analytical consumption.

Fact tables:

```text
fact_productivity_interval
fact_productivity_monthly
fact_productivity_daily
fact_productivity_hourly
fact_productivity_activity
fact_source_file_audit
```

Dimension tables:

```text
dim_date
dim_month
dim_hour
dim_activity_category
dim_source_file
```

## Final Serving Output

The Gold Delta folders are exposed through Synapse Serverless SQL views in the database:

```text
productivity_monitor_db
```

Synapse workspace:

```text
syn-productivitymonitor-dev
```

Serverless SQL endpoint:

```text
syn-productivitymonitor-dev-ondemand.sql.azuresynapse.net
```
